Notes on Uniform Resource Identifiers

The ultimate reference (maybe?): https://datatracker.ietf.org/doc/html/rfc3986

RFC 3986 gives some advices on several pitfalls:

  1. A 'resource' pointed by a URI does not necessarily have to be an electronic document of some kind; it can literally be anything. (e.g., human beings, corporations, books, math equations, etc.)
  2. An identifier does not necessarily define or embody the resource by itself (but it could; e.g., query strings)
  3. The system that uses the identifier does not necessarily access the resource pointed by that identifier.
  4. An identifier may be pointing to multiple resources (e.g., DNS-based load balancing), and the resource it points to might change (e.g., DNS registry changes).
